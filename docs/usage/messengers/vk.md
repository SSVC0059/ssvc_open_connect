# Уведомления VK (Open Connect)

Раздел **Настройки OC → Мессенджеры** доступен при сборке с флагом `FT_VK_BOT=1` (в проекте по умолчанию включён в `features.ini`). Устройство вызывает методы раздела **Messages** через HTTPS `https://api.vk.com/method/…` с ключом доступа сообщества и параметром `v` (версия API, в настройках по умолчанию `5.199`).

Справочник методов: [Описание методов API](https://dev.vk.com/ru/method). Конкретно для отправки текста: [messages.send](https://dev.vk.com/ru/method/messages.send), правка: [messages.edit](https://dev.vk.com/ru/method/messages.edit), пост на стене: [wall.post](https://dev.vk.com/ru/method/wall.post).

Гайды по чат-боту сообщества (отправка, лимиты, `random_id`, клавиатура): [Сообщения](https://dev.vk.com/ru/api/bots/development/messages), [Клавиатура](https://dev.vk.com/ru/api/bots/development/keyboard).

## Ключ доступа сообщества (токен)

Нужен **ключ доступа сообщества** (community access key), с которым сервер VK разрешает вызывать нужные методы от имени группы.

1. **Где создать ключ** — в интерфейсе ВКонтакте: **Управление сообществом** → **Настройки** → **Работа с API** → создание ключа с нужными возможностями. Общая логика ключей и типов токенов изложена в официальном разделе: [Ключи доступа — общие сведения](https://dev.vk.com/ru/api/access-token/getting-started).  
   Дополнительно по сценарию «сообщения сообщества» см. материалы раздела **API** на [dev.vk.com](https://dev.vk.com/ru) с поиском по запросам вроде *«сообщения сообщества»*, *«ключ доступа»* — точные подразделы интерфейса документации VK периодически переносятся, канал **dev.vk.com** остаётся источником правды.

2. **Права (scopes)** для Open Connect:
   - **`messages`** — обязательно для `messages.send` / `messages.edit`;
   - **`wall`** — если используете публикацию итога ректификации через `wall.post` (владелец записи — сообщество, см. описание метода [wall.post](https://dev.vk.com/ru/method/wall.post) и параметры `owner_id` / `from_group` в прошивке).

3. **Не путать с пользовательским токеном** — для сценария «бот сообщества пишет пользователям» нужен именно **ключ сообщества**, а не отдельный пользовательский OAuth-токен с `vk.com/oauth` (см. разделы про виды токенов в [документации по ключам доступа](https://dev.vk.com/ru/api/access-token/getting-started)).

4. **Длина токена** — у современных ключей вид `vk1.a.…` строка длинная; в Open Connect она сохраняется целиком. Не обрезайте при копировании.

## Параметр `peer_id` (официально: метод `messages.send`)

В методе [messages.send](https://dev.vk.com/ru/method/messages.send) задаётся обязательный параметр **`peer_id`** — идентификатор **чата назначения** в модели переписок VK (см. описание поля в справочнике метода на **dev.vk.com**).

Ниже — типичные случаи для настройки Open Connect (отправка **от имени сообщества**):

| Сценарий | Что подставить в поле peer_id в OC |
| -------- | ---------------------------------- |
| Личное сообщение **пользователю** | **Положительное** число — [идентификатор пользователя](https://dev.vk.com/ru/method/users.get) (`user_id`). Это же значение используется как `VK_PEER_ID` в `example/.env`. |
| **Беседа** (мультичат) | Обычно **`2000000000 + chat_id`**, где `chat_id` — идентификатор беседы в API сообщений (см. описание `peer_id` у [messages.send](https://dev.vk.com/ru/method/messages.send) и объекты переписок, например [conversation](https://dev.vk.com/ru/reference/objects/conversation)). `chat_id` можно получить из ответов методов `messages.*` (список диалогов и т.д.) — см. справочник методов. |

**Важно:** пользователь должен иметь возможность получать сообщения от сообщества (настройки приватности и первый контакт с сообществом). Иначе API вернёт ошибку (часто обсуждаемая в связке с сообществами код **901** — см. сообщения об ошибках в ответе `execute`/`messages.send` в [документации по ошибкам API](https://dev.vk.com/ru/reference/errors)).

**Как узнать свой числовой `user_id`:** официально — метод [users.get](https://dev.vk.com/ru/method/users.get) (при необходимости с `user_ids` или из контекста авторизации приложения). Внешние подсказки и сторонние боты для отображения id не являются официальной справкой VK.

## Что нужно в VK (краткий чек-лист)

1. Сообщество с **включёнными сообщениями** и при необходимости режимом для **чат-бота / возможностей ботов**, если используете inline-клавиатуру (см. сообщения об ошибке **912** в ответах API).
2. **Ключ доступа** с правами **`messages`**, при стене — **`wall`**.
3. До **трёх** числовых **`peer_id`** в интерфейсе OC: живой статус, тревоги, итог (стена — отдельно через `group_id` и `wall.post`).

## Настройка в интерфейсе OC

1. Откройте **Настройки OC → Мессенджеры**.
2. Включите подсистему **VK** (если отключена в списке подсистем).
3. В блоке VK укажите **токен** и **`peer_id`** для каналов (для живого статуса — тот же числовой id, что **`VK_PEER_ID`** в `example/.env`, если это личный диалог с пользователем).
4. Поле **«Кнопка „Веб“»** (`keyboard_base_url`): **оставьте пустым**, чтобы не отправлять inline-клавиатуру (как `example/vk_bot.py`). Если указать `http(s)://…`, в сообщение добавится кнопка `open_link`; для этого VK может потребовать режим **чат-бота** в настройках сообщества (иначе возможна ошибка **912**).
5. При необходимости включите **публикацию на стене** для итога.
6. Сохраните настройки.

Конфигурация хранится на устройстве (`/config/vk.json`); REST: `GET`/`PUT` `/rest/vk/config` (при отключённой подсистеме ответ **423**).

## Поведение на устройстве

- Отправка и правка сообщений выполняется **в отдельной задаче FreeRTOS**, привязанной к **ядру 1**, с неблокирующим циклом.
- **Живой статус** периодически обновляется через `messages.edit` (при ошибке возможна повторная отправка). Текст оформляется через параметр **`format_data`** (жирные фрагменты: заголовок «OpenConnect», подписи «Этап:» / «TP1:» / «TP2:», названия зон); смещения считаются в **UTF-16**, как ожидает VK. Если API вернёт ошибку на `format_data`, обновите версию API в настройках или сообщите — можно будет отключить разметку.
- **Тревоги** ставятся в небольшую очередь фиксированного размера.
- По **завершении ректификации** при включённой опции выполняется `wall.post` с кратким текстом по метрикам/датчикам (без графиков и временных рядов).

## Ограничения

- Inline-клавиатура: только кнопка **ссылка** (`open_link`), без Long Poll и callback на ESP32. Пока URL кнопки **пустой**, клавиатура **не отправляется**. Формат параметра `keyboard` и типы кнопок — в официальном разделе [Клавиатура (чат-боты)](https://dev.vk.com/ru/api/bots/development/keyboard); при неверном JSON возможна ошибка **911** (*Keyboard format is invalid*). Включение клавиатуры в интерфейсе сообщества и пункт **«Возможности ботов»** описаны там же; без этого возможна ошибка **912**. После включения кнопки `open_link` при ошибке **912** проверьте настройки сообщества.
- Для стабильной работы нужен сетевой доступ устройства к `https://api.vk.com`.
