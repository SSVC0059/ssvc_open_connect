# Регистрация ошибки после падения контроллера в дамп

!!! warning "Важная информация"
    Полноценная работа с дампами реализована в версии прошивки OpenConnect 0.2.5.2. Размер собранного файла `Coredump.bin` должен быть 64кбт
    Дамп размером в 32кбт не принимается для расследования - обновитесь на актуальную версию прошивки


Если ваше устройство перезагрузилось или прекратило работу некорректно, нам может потребоваться «дамп памяти» (Core Dump) для поиска причины. Это «снимок» состояния системы в момент сбоя.


## 1. Когда нужно отправлять дамп?
Не каждую проблему нужно расследовать через дамп. Присылайте его, если:

- Устройство внезапно перезагрузилось во время работы.
- Произошло «зависание»: экран не реагирует, но устройство включено.
- Ошибка повторяется при выполнении одного и того же действия (например, при запуске конкретного профиля).
- Техподдержка или разработчик явно попросили прислать файл дампа.


2. Что нужно подготовить перед отправкой
Для успешного расследования нам критически важны два компонента:

Точная версия прошивки:

 - Посмотрите версию в меню устройства или в веб-интерфейсе (например, v2.1.0).

 - Если вы собирали прошивку сами, укажите дату сборки или хэш из Git.

Файл дампа (.bin):

 - Выгрузите файл из памяти устройства через веб-интерфейс

## 3. Как правильно оформить сообщение
Пожалуйста, не присылайте файл без описания. Чтобы мы быстрее нашли ошибку, укажите:

**Что произошло:** (например: «Устройство ушло в перезагрузку в середине процесса перегона»).

**Ваши действия:** Что вы нажали или какой режим работал в этот момент?

**Настройки:** Если используется специфический профиль, приложите скриншот настроек или JSON-конфиг. 

**Оформление ошибки** Как правильно оформлять ошибку написано в [соответствующем разделе](http://127.0.0.1:8000/ssvc_open_connect/faq/check_list/#_2)

**Важно:** Файл дампа бесполезен без информации о версии прошивки. Если вы обновили прошивку и произошел сбой — указывайте именно ту версию, которая была установлена в момент падения.


## 4. Куда отправлять?
Отправьте собранную информацию на электронную почту `nl.ruslan@yandex.ru`:

**Тема:** «Дамп. Версия OpenConnect [ваша версия], Версия SSVC: [ваша версия]».

**Вложения:**

- Файл coredump.bin.
- Требуемые скриншоты и при необходимости конфигурационные файлы
- Прочие файлы полезные для расследования

**Описание ситуации.**

- Оформленная ошибка согласно требованиям описанным в [соответствующем разделе](http://127.0.0.1:8000/ssvc_open_connect/faq/check_list/#_2)
