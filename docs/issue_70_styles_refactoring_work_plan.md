# План работ по Issue #70: Глобальный рефакторинг работы со стилями

План составлен на основе [Issue #70 — Глобальный рефакторинг работы со стилями](https://github.com/SSVC0059/ssvc_open_connect/issues/70).

---

## Цель

Привести архитектуру стилей фронтенд-приложения Interface (Svelte 5) к единообразию: устранить критические ошибки (несуществующие переменные, некорректный `rgba()`), консолидировать переменные и темы, унифицировать систему стилей и перенести глобальные классы в компонентные стили.

---

## Этап 1. Исправление критических ошибок (приоритет: высокий, 1–2 дня)

| № | Действие | Детали | Sub-issue |
|---|----------|--------|-----------|
| 1.1 | Добавить недостающие CSS-переменные | В `app.css`: алиасы `--glass-primary-*`, `--glass-secondary-*` → `var(--primary-*)`; добавить `--red-100` (например `#fee2e2`); добавить RGB-переменные для `rgba()`: `--primary-100-rgb`, `--primary-500-rgb`, `--black-rgb`, `--blue-500-rgb` | См. шаблон «Этап 1.1» |
| 1.2 | Исправить `rgba()` с CSS-переменными | В `_tabs.scss`, `_modals.scss`, `_mixins.scss`, `_inputs.scss`: заменить `rgba(var(--primary-*), alpha)` и аналоги на `rgba(var(--*-rgb), alpha)` или `color-mix(in srgb, var(--*) X%, transparent)` | См. шаблон «Этап 1.2» |
| 1.3 | Удалить дубликат normalize.css | Оставить один источник: `src/normalize.css`; удалить `src/lib/styles/normalize.css` | См. шаблон «Этап 1.3» |
| 1.4 | Унифицировать импорты в SCSS | Во всех SCSS-файлах использовать `@use "$lib/styles/base/mixins"` (и аналоги) вместо относительных путей типа `@use "../base/mixins"` | См. шаблон «Этап 1.4» |

**Критерий завершения этапа:** стили с `--glass-*` и `rgba()` работают корректно; один файл normalize; единообразные импорты.
