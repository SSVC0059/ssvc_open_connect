# Ответы на частые вопросы

## На странице телеметрии отображается сообщение" "Версия API SSVC не поддерживается. Обновите прошивку SSVC на актуальную
(Требуется версия API: 1.6, текущая: недоступна)"

<div style="display: flex; justify-content: center;">
    <img src="/ssvc_open_connect/faq/media/f5.png" style="height:auto"> 
</div>

### Если текущая версия API указано как недоступна, то:

- Проверьте правильность подключения [UART](https://ssvc0059.github.io/ssvc_open_connect/usage/interfaces/uart/)
- OpenConnect не должен включаться ранее чем SSVC0059_v2. Это связанно с архитектурными особенностями контроллера

### Если текущая версия API указана ниже ожидаемой:

Обновите прошивку SSVC0059_v2 до актуально со страницы релизов [производителя](https://smartmodule.ru/portfolio/0059_v2/). 


## В мастере старта ректификации, после нажатия кнопки запустить, окно закрывается но на ssvc процесс не стартует

1. Убедитесь, что у вас установлена актуальная прошивка openconnect, а так же нет предупреждения, что версия API не соответствует требуемой
2. Убедитесь, что в меню настроект SSVC0059_v2 установлен флаг удаленный запуск. Вход в меню настроек SSVC осуществляется удержанием центральной кнопки при включении SSVC0059_v2

</div>

## <u>SSVC не реагирует на команды или выдается ошибка при сохранении настроек</u>

1. В первую очередь нужно проверить надежность и соединения UART. Если идет телеметрия, а команды нет - проверьте правильность подключения пина TX ESP32
2. Проверьте прошивку SSVC. На старых прошивках часть команд может не работать. Всегда обновляйтесь на актуальную версию


## <u>Какие требования к прошивке SSVC?</u>

Для полноценной работы требуется SSVC0059_v2 с версией API не ниже заявленной в релизе
Подходящие релизы SSVC можно найти на странице релизов на [официальном сайте](https://smartmodule.ru/portfolio/0059_v2/)

Переходим в раздел `Подготовка -> Автоматическая установка актуальной версии` 

В самом конце страницы находим кнопки прошивки устройства
