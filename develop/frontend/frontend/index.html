    📑 SCSS Классы и Компоненты Пользовательского Интерфейса

    Данный документ описывает основные SCSS-классы и миксины, используемые для построения интерфейса приложения. Стили основаны на концепции Glassmorphism и включают адаптивные решения для мобильных устройств, а также поддержку Темной темы (Dark Mode).

    ✨ Миксины (Mixins)

    Миксин Назначение Используемые файлы
    @mixin glassmorphism Применяет базовые стили стекломорфизма к элементу: фон с прозрачностью (var(--glass-background)), рамка (var(--glass-border)) и тень (var(--glass-shadow)). mixins.scss
    @mixin flex-center Утилитарный миксин для центрирования содержимого по вертикали и горизонтали. mixins.scss
    @mixin transition-all Применяет общую переменную для плавных переходов (var(--transition)). mixins.scss
    @mixin dark-theme-color Устанавливает цвет текста var(--primary-200) и других элементов специально для темной темы. mixins.scss
    @mixin compact-points Определяет стили для компактного отображения списков данных/точек (например, параметров) в виде таблицы с адаптивной сеткой. mixins.scss

    ⚙️ I. Настройки и Элементы Форм

    Класс Назначение
    .settings-container Основной контейнер страницы настроек. Использует glassmorphism, имеет ограничение по высоте (80vh) и прокрутку.
    .settings-panel Блок внутри настроек для группировки информации, стилизованный под полупрозрачную панель.
    .settings-section Отдельный раздел внутри панели настроек с заголовком и нижней границей.
    .settings-item Строка для одной настройки: обеспечивает выравнивание метки и элемента управления по горизонтали (justify-content: space-between).
    .settings-grid Контейнер для двухколоночной сетки панелей настроек на широких экранах.
    .input-field Универсальное поле ввода с базовыми стилями, эффектом фокуса и стилями для темной темы.
    .toggle-container / .toggle-slider Контейнеры для создания кастомного переключателя/тумблера (вкл/выкл) с поддержкой состояний :checked и :disabled.
    .error-message / .range-error Стили для отображения сообщений об ошибках валидации.
    .settings-description Маленький текст серого цвета для пояснения настройки.

    🧭 II. Вкладки (Tabs) и Навигация

    Класс Назначение
    .tabs-container Основной компонент для вкладок.
    .tabs-nav Десктопная навигация (скрыта на мобильных).
    .tab Отдельная вкладка в десктопной навигации (с состоянием .tab-active для активной вкладки).
    .mobile-tabs-header Мобильный компонент для навигации: отображает текущую вкладку и содержит кнопку-переключатель.
    .mobile-tabs-dropdown Выпадающий список вкладок для мобильных устройств, стилизованный под glassmorphism.

    💬 III. Модальные Окна и Wizard (Мастер Настройки)

    Класс Назначение
    .modal-dialog / .modal-content Основные стили для модального окна: центрирование, фон с блюром, контент-блок с glassmorphism.
    .modal-body Скроллируемая область контента модального окна.
    .modal-actions Контейнер для кнопок действий в нижней части модального окна.
    .wizard Основной контейнер Мастера Настройки. Обеспечивает большую фиксированную форму, включающую шаги и навигацию.
    .steps-container / .step-circle Элементы для визуализации текущего шага в мастере.
    .wizard-nav Нижняя панель навигации Wizard'а с кнопками "Назад" и "Далее/Сохранить".

    🌡️ IV. Датчики и Управление Зонами

    Класс Назначение
    .sensors-main-container Контейнер страницы, отображающей все зоны и датчики.
    .zone-panel Панель, представляющая отдельную зону (например, "Теплые полы", "Бассейн").
    .sensors-grid Адаптивная сетка для отображения карточек датчиков внутри зоны.
    .sensor-card Карточка отдельного датчика (например, DS18B20), отображающая его адрес, текущую температуру и настройки.
    .temperature-bar-compact Визуальный индикатор (шкала) для отображения текущей температуры и порогов.
    .warning-marker / .critical-marker Маркеры на шкале, указывающие на пороговые значения (предупреждение/критический).
    .threshold-editor Компонент для редактирования пороговых значений температуры.

    🚀 V. Общие Утилиты

    Класс Назначение
    .btn / .btn-primary / .btn-success Базовые стили для кнопок и их модификаторов (основной, успешное действие и т.д.).
    .loading-container / .loading-spinner Стили для состояний загрузки (отображение текста и анимированного спиннера).
    .error-container Контейнер для отображения сообщений об ошибках (например, при загрузке данных).
    .disabled-overlay Полупрозрачное наложение, которое блокирует взаимодействие с компонентом, когда он отключен (например, для компонента Telegram).
